1. 背景与目标

1.1 项目背景

LabTrace 目前的核心功能是处理实验室检验报告（如血常规、生化检查），通过 OCR 识别将纸质报告数字化。但用户在日常健康管理中还会产生大量影像检查报告，如 CT、MRI、X光、超声、心电图等，这些报告同样需要统一管理和历史追踪。

example/ 目录下已有部分影像报告的 JSON 格式数据样本，表明系统需要扩展以支持影像检查报告的导入、存储和查看。

1.2 核心目标

  • 统一入口：在现有 OCR 上传页面增加报告类型选择（实验室/影像）
  • 分类处理：针对影像报告采用不同的 OCR 解析策略，提取关键字段
  • 影像查看：支持 PDF、图片格式的查看，主要以原始影像为主
  • 结构化存储：提取诊断结论、检查部位、影像表现等关键信息
核心价值

扩展 LabTrace 从"检验报告管理"升级为"完整健康档案管理"，覆盖用户的主要医疗检查数据类型。

3.4 模块3：影像OCR解析器 P1

3.4.1 功能描述

影像报告的 OCR 解析重点不同于实验室检验报告。影像报告通常是文档格式，重点提取：

  • 检查部位（如：胸部、腹部、头部）
  • 影像表现（如：肺纹理增粗、右肺结节）
  • 诊断结论（如：未见明显异常、右肺磨玻璃结节）

3.4.2 解析策略

OCR结果解析流程： 1. 遍历所有OCR文本块 2. 关键词匹配： - "检查部位" / "检查部位：" → 提取部位 - "影像表现" / "所见" / "描述：" → 提取表现文本 - "诊断结论" / "印象" / "印象诊断" → 提取诊断结论 3. 多行合并：将同一字段的多行文本合并 4. 结构化输出

3.4.3 解析字段

字段名 来源 说明 OCR提取关键词
exam_site OCR提取 检查部位 检查部位、检查部位：
exam_description OCR提取 影像表现/所见 影像表现、所见、描述：
diagnosis_result OCR提取 诊断结论 诊断结论、印象、印象诊断：
report_type 用户选择 影像类型 -
exam_item_name OCR提取/手动 检查项目名 （从文件名或报告标题提取）

7.2 OCR解析实现

// ParseImagingReport 解析影像报告OCR结果
func ParseImagingReport(ocrResults []OCRResult) *ImagingParsedResult {
    result := &ImagingParsedResult{}

    for _, block := range ocrResults {
        text := strings.TrimSpace(block.Text)
        lowerText := strings.ToLower(text)

        // 匹配检查部位
        if strings.Contains(lowerText, "检查部位") {
            result.ExamSite = extractAfterColon(text)
            continue
        }

        // 匹配诊断结论（重点）
        if strings.Contains(lowerText, "诊断结论") ||
           strings.Contains(lowerText, "印象：") ||
           strings.Contains(lowerText, "印象诊断") {
            result.DiagnosisResult = extractAfterColon(text)
            continue
        }

        // 匹配影像表现
        if strings.Contains(lowerText, "影像表现") ||
           strings.Contains(lowerText, "所见") {
            result.ExamDescription = extractAfterColon(text)
            continue
        }
    }

    return result
}

11. 总结

本次需求将 LabTrace 从单一的"检验报告管理"扩展为"完整健康档案管理"，主要实现：

  1. 统一入口：在 OCR 上传页面支持报告类型选择
  2. 分类处理：影像报告采用不同的 OCR 解析策略
  3. 结构化存储：提取诊断结论、检查部位等关键信息
  4. 专业查看：PDF/图片查看器支持翻页、缩放
  5. 数据分离：独立的影像报告表，与检验报告分离管理
注意事项
  • 影像报告的 OCR 识别准确度依赖于报告的格式规范程度
  • 部分医院报告可能包含手写内容，OCR 识别困难
  • 建议用户优先使用 PDF 格式报告，图片次之

所有功能开发完成后需要进行完整的端到端测试，确保用户体验流畅。
