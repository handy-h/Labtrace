批量 OCR 上传功能需求

溯源健康 (LabTrace) - 个人纵向检化验管理平台

版本：V1.0 状态：已实现 日期：2026-05-12 关联模块：检验单上传与 OCR

1. 背景与目标

1.1 问题描述

用户在管理个人健康数据时，往往需要一次性上传多张检验报告单（图片或 PDF）。此前系统仅支持单文件上传，用户必须逐个选择文件并点击上传按钮，操作繁琐且效率低下。

1.2 设计目标

核心目标：允许用户在文件选择器中一次性选择多个图片或 PDF 文件，系统依次自动提交 OCR 识别，并提供可视化的批量上传进度反馈。
  • 批量选择：支持在文件选择对话框中同时选择多个文件（图片/PDF 混合）
  • 顺序提交：文件按选择顺序依次上传，每个文件独立调用 OCR 接口
  • 进度可视：实时显示上传进度（当前第几个/共几个）和每个文件的状态
  • 错误容忍：单个文件上传失败不影响后续文件，失败文件可单独重试

2. 功能规格

2.1 文件选择

属性 规格
选择方式 标准文件选择对话框，支持 multiple 属性
支持格式 图片（image/*）和 PDF（.pdf），可混合选择
文件列表显示 选择 2 个及以上文件时，显示文件列表（序号、文件名、大小、删除按钮）
文件移除 用户可逐个移除已选择的文件，无需重新打开文件对话框

2.2 上传按钮状态

状态 按钮文本 按钮状态
空闲 上传 可点击
单文件上传中 上传中... 禁用
批量上传中 正在上传 (2/5) 禁用

2.3 批量上传队列组件

位于页面左下角（与 OCR 配额条叠加显示），固定定位。

组件结构：
  • 标题栏："批量上传队列" + "N/M 已完成" 计数
  • 进度条：蓝色进度条显示整体完成百分比
  • 文件列表：每个文件显示状态图标 + 文件名
    • ○ 待上传（灰色空心圆）
    • ● 上传中（橙色实心圆）
    • ✓ 已完成（绿色勾）
    • ✗ 失败（红色叉）

2.4 上传逻辑

用户选择文件 ↓ 点击"上传"按钮 ↓ ┌─ 单文件？─→ 使用原有上传逻辑（保持兼容） │ └─ 多文件？─→ 快照当前表单字段（受检者/医院/采样日期） ↓ ┌─ for 循环每个文件 ─────────────────────────────┐ │ ↓ │ │ 设置状态为"上传中" │ │ ↓ │ │ 构造 FormData 并调用 API │ │ ↓ │ │ 成功？→ 标记为"已完成" │ │ 失败？→ 标记为"失败"，记录错误信息 │ │ ↓ │ └─────────────────────────────────────────────────┘ ↓ 刷新报告列表 ↓ 显示汇总提示（X 成功，Y 失败）

2.5 表单字段快照

重要：批量上传开始时，系统会快照当前表单字段（受检者、医院、采样日期）。后续所有文件均使用快照值，即使用户在上传过程中修改了表单字段也不受影响。

2.6 边界情况处理

场景 处理方式
文件重复（MD5 冲突） 该文件标记为"失败"，继续处理后续文件
网络错误 该文件标记为"失败"，记录错误信息，继续后续文件
OCR 配额耗尽 服务器返回错误，该文件标记为"失败"，继续后续文件
用户修改表单字段 不影响进行中的上传（使用快照值）
用户重新选择文件 仅在非上传状态时重置队列，上传中时新文件保存但不影响当前批次
上传中按钮再次点击 按钮处于禁用状态，无法点击
必填字段缺失 弹出提示"请选择文件和受检者"，不执行上传

3. 用户界面

3.1 上传表单区域

位于页面顶部的卡片区域，包含以下字段：

  • 受检者下拉框（必填）
  • 医院下拉框（选填）
  • 采样日期选择器（选填）
  • 文件选择器（支持多选，显示已选文件列表）
  • 上传按钮（动态文本）

3.2 已选文件列表

当选择 2 个及以上文件时，在文件选择器下方显示：

  • 最大高度 6rem，超出可滚动
  • 每行显示：序号、文件名（截断）、文件大小、删除按钮
  • 字体大小 0.75rem，紧凑布局

3.3 批量上传队列组件

  • 位置：固定定位，左下角（bottom: 0.75rem, left: 0.75rem）
  • 层级：z-index: 50（与 OCR 配额条相同层级，自然叠加）
  • 宽度：最小 14rem，最大 20rem
  • 出现条件：batchQueue.length > 0 时显示
  • 消失条件：用户重新选择文件时重置队列

4. 技术实现

4.1 前端实现

变更点 说明
文件输入元素 添加 multiple 属性
数据模型 selectedFile → selectedFiles（数组）
批量状态追踪 新增 batchQueue、batchUploading、batchCurrent 响应式变量
上传函数 重写为 async/await，支持单文件/多文件分支
计算属性 新增 batchDoneCount、batchPct 用于进度显示

4.2 后端兼容性

无需后端变更：现有 POST /api/v1/ocr/upload 接口已支持单文件上传，批量上传通过前端循环调用实现，每个文件独立处理。

4.3 缓存控制

前端 JS 文件通过 ?v=N 参数控制浏览器缓存。功能更新后需递增版本号（如 v=2 → v=3）。

5. 用户流程示例

5.1 典型批量上传流程

  1. 用户选择受检者"张三"、医院"市第一人民医院"、采样日期"2026-05-01"
  2. 点击文件选择器，在文件对话框中选择 3 张图片（血常规、肝功能、肾功能）
  3. 页面显示文件列表：#1 blood_test.jpg (120KB)、#2 liver.jpg (95KB)、#3 kidney.jpg (88KB)
  4. 点击"上传"按钮，按钮文本变为"正在上传 (1/3)"
  5. 左下角出现批量上传队列组件，显示 3 个文件状态
  6. 第 1 个文件上传成功，状态变为 ✓，按钮文本变为"正在上传 (2/3)"
  7. 第 2 个文件上传成功，状态变为 ✓，按钮文本变为"正在上传 (3/3)"
  8. 第 3 个文件上传成功，状态变为 ✓，批量上传完成
  9. 弹出提示："批量上传完成：3 成功"
  10. 报告列表自动刷新，显示 3 条新记录

5.2 单文件上传（向后兼容）

当用户仅选择 1 个文件时，系统使用原有上传逻辑，行为与更新前完全一致。

6. 测试验证

测试场景 预期结果
选择单个文件上传 行为与原有功能一致，按钮显示"上传中..."
选择 3+ 文件批量上传 依次上传，按钮显示进度，队列组件显示状态

批量上传完成后查看队列 队列保持显示，可查看每个文件的最终状态
