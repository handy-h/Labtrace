界面计划

溯源健康 (LabTrace) 界面设计计划

Overview

  • 该计划描述一套面向 PC（WebApp，1440px 宽默认画布）的静态界面，覆盖“溯源健康 (LabTrace)”的核心工作流与管理页面。设计目标是提升检验单数字化校核效率、实现多机构数据纵向合并与动态参考区间可视化，并保证数据私有化与备份流程清晰可操作。
  • 计划包含 6 个屏幕：仪表盘、检验单上传与渐进式 OCR 预览（沉浸式比对视图）、受检者档案列表与详情、检验项目库与别名/单位映射管理、纵向趋势分析、设置（备份与密钥管理）。示例数据已内置用于占位与展示。

Plan Details

应用信息（元数据）

  1. 应用类型：WebApp（PC 优先）。
  2. 画布宽度参考：1440px。
  3. 屏幕列表（共 6 个）：
    • 仪表盘 / 快速看板（异常筛选）
    • 检验单上传与渐进式 OCR 预览 + 规则记忆（沉浸式比对视图）
    • 受检者档案列表与详情（含年龄自动计算）
    • 检验项目库与别名/单位映射管理（含计算校验规则界面）
    • 纵向趋势分析（合并多源、动态参考带、下钻）
    • 设置 / 数据备份与密钥管理
  4. 界面语言：简体中文。
  5. 示例数据规模（用于占位与示范）：
    • 受检者 1：姓名 “张三”，检验记录 8 条；
    • 受检者 2：姓名 “李四”，检验记录 11 条。
    • 示例记录包括不同医院来源、不同单位与年龄对应的参考区间，用于演示别名映射、单位转换和动态参考带。
  6. 视觉方向（高层）：中性专业、信息密集型布局。主色为蓝灰系，异常提示色使用橙/红（浅橙用于中信心提示、浅红用于低信心高亮）。

1. 仪表盘 / 快速看板（顶部入口）

目的：提供数据概览和异常筛选入口，快速定位需要人工核效的检验数据。布局自上而下：

  1. 顶栏（全局）
    • 应用 logo + 名称（左）、全局搜索框（中）、同步/导入状态指示、用户偏好与帮助（右）。
  2. 左侧侧栏（垂直导航）
    • 导航项：仪表盘（高亮）、上传 OCR、受检者、项目库、趋势分析、设置。可折叠。
  3. 主内容区（仪表盘）
    • 快捷看板（卡片式）：总受检者数、待核效条目数、异常条目数、最近导入医院数量。
    • 异常筛选栏（水平）：按医院、时间范围、项目类别、置信度区间（≥95 / 80–94 / <80）、提示符（H/L/阴阳）过滤。
    • 结果摘要表（下方）：显示被筛选数据的分页表格，列包括：状态图标、受检者姓名、采样日期、检验项目、当前结果（高亮）、医院来源、置信度（数值与视觉化）、提示符、操作（查看/下钻）。
  4. 右侧快速面板（可展开）
    • 选中某行时显示该条项的快速预览：小尺寸纵向趋势迷你图、最近三次原始凭证缩略图、快速备注。

2. 检验单上传与渐进式 OCR 预览（沉浸式比对视图）

目的：支持用户上传图片/PDF，完成全量 OCR、人工映射列、记忆医院规则并入库。布局自上而下/左右联动：

  1. 顶部上传区
    • 上传控件（文件拖拽 + 浏览），支持多文件批量上传，上传队列与每个文件的识别状态（待识别/识别中/待核效/已入库）。
  2. 主视图分两栏（左：原始凭证，右：数据网格）
    • 左侧原始凭证查看器（占宽约 55%）：大图预览、放大镜工具、缩放快捷（Space），高亮呼吸框（当右侧选中行时自动在图片上标注对应区域并放大局部）。
    • 鼠标框选功能：允许用户在图片上框选列头/列区域以告诉系统哪些图像区域对应哪一列（医院规则学习触发器）。
    • 右侧数据网格（占宽约 45%）：
      • 表格列：状态图标、检验项目（冻结列）、当前结果、单位、置信度（%）、生物参考区间、纵向趋势迷你图、备注、操作。表格支持行内编辑、单元格聚焦。
      • 置信度视觉规则：高信心 ≥95% 标准黑字；中信心 80–94% 文字下方浅橙虚线；低信心 <80% 单元格背景浅红高亮，自动获取焦点并弹出校正输入框。
      • 自动校验与提示：若入库前检测到重复 MD5（文件或条目），显示重复提示并阻止重复入库按钮可用。
  3. 医院规则记忆条
    • 当用户完成列映射后，出现“保存为该医院规则”的按钮，规则预览显示列名映射、单位转换、备注列位置。
    • 规则列表入口：显示已保存医院规则（按医院名称或样例文件预览）。
  4. 操作区（底部）：批量“标记已确认”、“入库”按钮；入库前展示“计算勾稽校验”结果摘要。

2.1 自定义列映射功能（增强）

目的：对于 OCR 识别结果不准确或格式不规范的检验单，提供精细化的三步式列映射工具，提升数据提取准确率。

入口：在报告状态为“待核效”时，列表页新增“自定义映射”按钮（紫色），点击后打开三步向导。

  • Step 1：框选表格区域
    • 左侧显示原始图像（或 OCR 块分布图），上方/右侧提示“请框选表格区域”。
    • 系统自动识别表格区域，用户可拖拽调整框选范围。
    • 若为 PDF 且 OCR 返回纯文本（无坐标信息），显示降级界面并提供跳转到“直接配置表头映射”。
  • Step 2：表头列角色映射
    • 右侧展示表头卡片网格，每张卡片对应一个检测列。
    • 下拉选项：检测项目、结果数值、单位、参考范围、备注、忽略。
    • 必填校验：必须指定且仅限一个为“检测项目”和“结果数值”。
  • Step 3：单元格级修复
    • 智能网格视图，保留原始 OCR 文本悬浮提示。
    • 分割：选中单元格后按空格/冒号等分割，并预览拆分结果。
    • 合并：选中多行时显示警告并建议是否拆分为多行。
    • 支持 Ctrl+Z 撤销。

完成向导后调用相应 API 重新解析并返回 items。

3. 受检者档案列表与详情（含年龄自动计算）

目的：管理受检者基础信息与查看其历史检验记录。

  • 列表视图包含姓名、性别、出生日期、精确年龄、最近检验日期、记录数量、操作。支持搜索与排序。
  • 详情面板显示受检者基本信息、当前精确年龄、检验记录时间轴/表格、快速操作。

4. 检验项目库与别名/单位映射管理（含计算校验规则）

  • 核心统计条显示项目总数、别名数量、单位转换规则数量、计算校验规则数量。
  • 标准项目表包含项目编码、标准名称、常见别名、默认单位、常用单位示例、操作。
  • 别名映射管理器支持搜索并快速绑定医院项目到标准项目。
  • 单位标准化引擎界面展示源单位、目标单位、转换公式和示例验证。
  • 计算校验规则管理支持公式编辑、阈值设置、示例验证。

5. 纵向趋势分析（合并多源、动态参考带、下钻）

  • 筛选与图例栏支持受检者、项目、时间范围、医院、参考区间等过滤。
  • 主图为折线图，点上悬浮显示采样日期、医院、数值、单位、置信度、提示符。
  • 动态参考带根据年龄阶梯变化呈阶梯状色块。
  • 迷你表格显示数据点源信息和下钻操作。
  • 点下钻浮窗展示原始检验报告单缩图、备注和计算校验结果。

6. 设置 / 数据备份与密钥管理

  • 密钥管理区显示当前加密配置位置，并允许编辑与确认。
  • 备份与恢复区支持一键导出加密备份、导入 .bak 文件、显示验证结果。
  • 备份历史表列出备份时间、文件名、描述和操作。
  • 日志与审计支持按时间和操作类型过滤。

示例数据与占位指引

  • 受检者 A：男，出生日期 1978-03-12，示例检验记录 8 条。
  • 受检者 B：女，出生日期 1990-09-05，示例检验记录 11 条。
  • 示例数据应包含低置信度和中置信度场景。

  • 数据网格中“检验项目”列固定。
