## Overview

- 该计划描述一套面向 PC（WebApp，1440px 宽默认画布）的静态界面，覆盖“溯源健康 (LabTrace)”的核心工作流与管理页面。设计目标是提升检验单数字化校核效率、实现多机构数据纵向合并与动态参考区间可视化，并保证数据私有化与备份流程清晰可操作。
- 计划包含 6 个屏幕：仪表盘、检验单上传与渐进式 OCR 预览（沉浸式比对视图）、受检者档案列表与详情、检验项目库与别名/单位映射管理、纵向趋势分析、设置（备份与密钥管理）。示例数据已内置用于占位与展示。

## Plan Details

### 应用信息（元数据）

1. 应用类型：WebApp（PC 优先）
2. 画布宽度参考：1440px
3. 屏幕列表（共 6 个）：
   - 仪表盘 / 快速看板（异常筛选）
   - 检验单上传与渐进式 OCR 预览 + 规则记忆（沉浸式比对视图）
   - 受检者档案列表与详情（含年龄自动计算）
   - 检验项目库与别名/单位映射管理（含计算校验规则界面）
   - 纵向趋势分析（合并多源、动态参考带、下钻）
   - 设置 / 数据备份与密钥管理
4. 界面语言：简体中文
5. 示例数据规模（用于占位与示范）：
   - 受检者 1：姓名 “张三”，检验记录 8 条；
   - 受检者 2：姓名 “李四”，检验记录 11 条。
   - 示例记录包括不同医院来源、不同单位与年龄对应的参考区间，用于演示别名映射、单位转换和动态参考带。
6. 视觉方向（高层）：中性专业、信息密集型布局。主色为蓝灰系，异常提示色使用橙/红（浅橙用于中信心提示、浅红用于低信心高亮）。

### 1. 仪表盘 / 快速看板（顶部入口）

目的：提供数据概览和异常筛选入口，快速定位需要人工核效的检验数据。
布局自上而下：

1. 顶栏（全局）
   - 应用logo + 名称（左）、全局搜索框（中）、同步/导入状态指示、用户偏好与帮助（右）。
2. 左侧侧栏（垂直导航）
   - 导航项：仪表盘（高亮）、上传 OCR、受检者、项目库、趋势分析、设置。可折叠。
3. 主内容区（仪表盘）
   - 快捷看板（卡片式）：总受检者数、待核效条目数、异常条目数、最近导入医院数量。
   - 异常筛选栏（水平）：按医院、时间范围、项目类别、置信度区间（≥95 / 80–94 / <80）、提示符（H/L/阴阳）过滤。
   - 结果摘要表（下方）：显示被筛选数据的分页表格，列包括：状态图标、受检者姓名、采样日期、检验项目、当前结果（高亮）、医院来源、置信度（数值与视觉化）、提示符、操作（查看/下钻）。
4. 右侧快速面板（可展开）
   - 选中某行时显示该条项的快速预览：小尺寸纵向趋势迷你图、最近三次原始凭证缩略图、快速备注。

### 2. 检验单上传与渐进式 OCR 预览（沉浸式比对视图）

目的：支持用户上传图片/PDF，完成全量 OCR、人工映射列、记忆医院规则并入库。
布局自上而下/左右联动：

1. 顶部上传区
   - 上传控件（文件拖拽 + 浏览），支持多文件批量上传，上传队列与每个文件的识别状态（待识别/识别中/待核效/已入库）。
2. 主视图分两栏（左：原始凭证，右：数据网格）
   - 左侧原始凭证查看器（占宽约 55%）
     - 大图预览、放大镜工具、缩放快捷（Space），高亮呼吸框（当右侧选中行时自动在图片上标注对应区域并放大局部）。
     - 鼠标框选功能：允许用户在图片上框选列头/列区域以告诉系统哪些图像区域对应哪一列（医院规则学习触发器）。
   - 右侧数据网格（占宽约 45%）
     - 表格列（自左向右）：状态图标（识别置信度视觉）、检验项目（冻结列）、当前结果（加粗）、单位、置信度（%）、生物参考区间（含适用范围）、纵向趋势迷你图、备注、操作。表格支持行内编辑、单元格聚焦。
     - 置信度视觉规则：
       - **高信心 (≥95%)**：标准黑字；
       - **中信心 (80–94%)**：文字下方显示浅橙色虚线提示复核；
       - **低信心 (<80%)**：单元格背景浅红高亮，自动获取焦点并弹出校正输入框，强制人工确认。
     - 自动校验与提示：若入库前检测到重复 MD5（文件或条目），显示重复提示并阻止重复入库按钮可用。
3. 医院规则记忆条
   - 当用户完成列映射（点击/框选列属性）后，出现“保存为该医院规则”的按钮，规则预览显示列名映射、单位转换、备注列位置。
   - 规则列表入口：显示已保存医院规则（按医院名称或样例文件预览）。
4. 操作区（底部）
   - 批量“标记已确认”、“入库”按钮；入库前展示“计算勾稽校验”结果摘要（显示偏差超过阈值的条目如：总蛋白偏差 >1 g/L 列出并弹出预警）。

### 3. 受检者档案列表与详情（含年龄自动计算）

目的：管理受检者基础信息与查看其历史检验记录。
布局自上而下与左右组合：

1. 列表视图（左侧或主区域）
   - 列表/表格列包括：姓名、性别、出生日期（支持日期选择器）、精确年龄（按每条检验的采样日期动态计算并显示年龄字段）、最近检验日期、记录数量、操作（查看详情/合并记录）。
   - 列表支持按姓名、出生日期、医院来源、最近检验日期搜索与排序。
2. 详情面板（右侧展开或点击打开独立详情页）
   - 受检者基本信息卡：姓名、性别、出生日期、当前精确年龄（数值 + 计算基准日期显示，例如“年龄（取样日）：45岁”）。
   - 检验记录时间轴/表格：按时间倒序列出所有检验记录（含医院、采样日期、项目摘要、置信度、提示符），每行含小尺寸“查看原始凭证”图标。
   - 快速操作：合并/拆分记录（用于处理重复受检者或误拆分）；导出该受检者备份包（加密 .bak）。

### 4. 检验项目库与别名/单位映射管理（含计算校验规则）

目的：维护标准化项目语义、别名映射、单位转换矩阵与计算勾稽规则。
布局自上而下：

1. 核心统计条（上部）
   - 当前标准项目总数、已映射别名数量、单位转换规则数量、计算校验规则数量。
2. 标准项目表
   - 列：项目编码、标准名称、常见别名（预览），默认单位、常用单位切换示例、操作（编辑别名/查看单位规则）。
3. 别名映射管理器（右侧或弹窗）
   - 搜索某医院项目识别结果，快速绑定到系统标准项目；支持批量绑定规则上传。
4. 单位标准化引擎界面
   - 单位转换矩阵展示：源单位、目标单位、转换公式、示例验证（界面显示示例输入与计算输出）。
   - 安全阀：在新增自定义公式时要求填写“源单位”和“目标单位”，并在界面右侧显示示例计算验证结果与示例输入框。
5. 计算校验规则管理
   - 规则列表（例如：总蛋白 = 白蛋白 + 球蛋白），每条规则显示示例、适用范围、警戒阈值。
   - 编辑/新增规则弹窗：公式编辑器（文本 + 可选择字段插入）、阈值设置（如偏差 >1 g/L 触发警告）、示例验证区显示若干示例数据计算结果。

### 5. 纵向趋势分析（合并多源、动态参考带、下钻）

目的：在统一坐标系下展示同一标准项目来自不同医院/时间的长期趋势，并可下钻查看原始凭证与备注。
布局自上而下/中间为主视图：

1. 筛选与图例栏（上部）
   - 受检者选择下拉、项目选择、时间范围、医院/来源多选、图例颜色说明、参考区间显示开关。
2. 主图（折线图）
   - 折线：每个医院为一条颜色区别的折线，点表示一次检验；点上悬浮显示：采样日期、医院、数值、单位、置信度、提示符标签。
   - **动态参考带**：图背景以色块表示参考区间，参考区间随年龄阶梯变化呈阶梯状带块；当该点的“采样当日年龄”对应不同参考区间时，背景区域应在对应 x 轴段显示相应色带。
   - Y 轴单位标签与自动缩放；若不同点源单位不同则在图上方显示“已统一单位为 X”（并在图注中标明原始单位来源）。
3. 迷你表格（下方或侧栏）
   - 数据点列表：按时间列出点的来源、原始值、已转换值、参考区间、提示符、操作（下钻）。
4. 下钻浮窗
   - 点击某数据点打开浮窗，展示：该条检验的原始检验报告单缩图（可放大）、完整备注、相关计算校验结果、按钮：在新窗口查看完整详情或跳至原始凭证在上传页的位置。

### 6. 设置 / 数据备份与密钥管理

目的：提供数据导出/导入、加密密钥配置与备份历史管理界面，保证本地私有化存储与恢复流程可见。
布局自上而下：

1. 密钥管理区
   - 显示当前加密配置位置示例（文本说明：.env 本地根目录），允许用户查看/编辑（编辑时需二次确认）。显示 SQLCipher 加密状态（已启用/未启用）。
2. 备份与恢复区
   - 一键导出加密备份（生成 .bak 文件）按钮，导出时要求填写备份描述。导入备份（选择本地 .bak）并显示验证结果（成功/失败、MD5 校验）。
   - 备份历史表：列出历史备份时间、文件名、描述、操作（下载/恢复/删除）。
3. 日志与审计
   - 基本操作日志列表（导入/导出/入库/规则编辑），支持按时间与操作类型过滤并导出审计记录。

### 示例数据与占位指引（用于设计填充）

- 受检者 A（张三）：男，出生日期 1978-03-12，示例检验记录 8 条，医院来源分布为医院甲（3 条）、医院乙（5 条）。记录包含血糖（mmol/L 与 mg/dL）、总蛋白/白蛋白/球蛋白（用于展示计算校验）、若干定性项目（阴/阳）。
- 受检者 B（李四）：女，出生日期 1990-09-05，示例检验记录 11 条，医院来源分布为医院乙（6 条）、医院丙（5 条）。记录包含血常规、肝功能项、数值跨单位场景与随年龄变动的参考区间示例。
- 示例数据应包含若干低置信度 (<80%) 的单元以展示红色强制校验场景，和若干中置信度（80–94%）以展示浅橙虚线提示。

## 交互说明（用于设计参考）

- 全局快捷键提示条：在界面右下角固定显示快捷键摘要（Tab: 切换单元格；Enter: 保存并跳转下一异常行；Space: 缩放图片）。
- 冻结列：在数据网格中“检验项目”列固定，便于横向滚动时参照。
- 规则记忆提示：在上传预览页明显显示“已为医院 X 保存映射规则（N 条）”的标签与“规则管理”入口。
