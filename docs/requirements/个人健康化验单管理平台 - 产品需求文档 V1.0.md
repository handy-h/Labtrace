

**个人健康化验单管理平台 - 产品需求文档 V1.0**


---

### 1. 文档概述

#### 1.1 文档目的
本文档旨在定义“个人健康化验单管理平台”的功能需求、业务流程及交互逻辑。该平台解决个人用户在不同医院间化验数据不互通、格式不统一、难以长期追踪健康趋势的痛点。通过OCR技术实现化验单的批量数字化录入，并提供标准化的数据维护与可视化工具。

#### 1.2 产品范围
- 核心功能：化验单上传、OCR智能识别、自定义解析规则配置、数据清洗与映射、趋势分析。
- 终端限制：仅提供Web端访问，不支持移动端适配。
- 用户体系：无注册/登录功能，通过“受检人”维度管理数据，实现数据隔离。

---

### 2. 用户角色与核心场景

#### 2.1 用户角色
- 数据管理者：负责上传化验单、配置医院解析规则、清洗数据、维护受检人档案的个人用户。

#### 2.2 核心场景
- **场景一：首次使用某医院化验单**
  用户选择受检人及医院 -> 上传化验单 -> 进入“自定义解析工具” -> 手动建立列映射关系（包含备注列） -> 设置单位转换公式 -> 配置参考范围及适用范围 -> 保存规则并导入数据。
- **场景二：复用已有规则**
  用户选择受检人及医院（系统自动匹配历史规则） -> 批量上传化验单 -> 系统自动识别并预览 -> 根据受检人年龄/性别自动匹配适用参考范围 -> 用户核对异常值 -> 一键入库。
- **场景三：跨医院数据趋势分析**
  用户查看“血糖”指标 -> 系统自动合并不同医院的同一项目数据 -> 统一单位后展示折线图 -> 根据当时受检人的年龄/性别动态标红异常区间。可以表格形式查看明细，展示受检时间、受检时年龄、数值、参考范围、异常标记、原始备注。

---

### 3. 产品架构与数据模型

#### 3.1 核心实体关系
- **受检人**：数据隔离的根节点，包含姓名、性别、出生日期（用于计算检测时的精确年龄）。
- **医院**：解析规则的绑定主体，不支持别名管理。
- **化验项目**：标准化的数据单元，包含名称、单位、参考范围组（支持多套适用范围，按性别/年龄维度区分）。
- **化验报告**：原始文件与识别结果的集合，关联受检人、医院、检测时间及整单备注。
- **数据行**：单条检测结果，包含数值/定性结果、单位、所用参考范围、异常标记、行级备注（镜检/文字性补充）。

#### 3.2 数据流向
原始文件（PDF/Img） -> OCR引擎 -> 原始识别文本 -> 解析规则引擎（列映射 + 单位转换 + 备注提取） -> 标准化数据 -> 本地数据库 -> 趋势图表（含动态参考范围匹配）。

### 4. 功能需求详情

#### 4.1 基础数据管理

##### 4.1.1 受检人管理
- 创建/编辑：录入姓名、性别、出生日期（必填，用于计算各历史检测点的精确年龄，以匹配参考范围）。
- 切换机制：顶部导航栏提供下拉菜单切换当前受检人，切换后全局数据视图即时刷新。
- 数据隔离：所有化验数据、规则配置均绑定当前选中的受检人ID。

##### 4.1.2 医院与规则管理
- 医院维护：支持新增、编辑医院名称。
- 规则库：系统按“医院”维度存储解析规则（列映射、单位转换公式、备注列映射）。

##### 4.1.3 化验项目库与参考范围
- 标准项目：系统预置常见项目（血糖、血脂等）。
- 动态新增：OCR识别到新项目时，触发“新项目确认”流程，强制用户补全单位与参考范围组。
- 项目别名：支持将不同医院的同类项目映射到标准项目（如“GLU”映射为“血糖”），确保趋势分析数据合并。
- **参考范围适用范围（新增）**
  - 每个参考范围不再全局唯一，而是绑定“适用范围”属性。
  - 适用范围维度：性别（男/女/全部）、年龄阶段（精确到岁，支持区间如18-49岁、≥50岁，或按天用于婴幼儿）。
  - 配置界面：在项目标准化配置时，可添加多条参考范围，每条均设置性别、年龄下限、年龄上限（留空表示无上限）。
  - 异常判定逻辑：读取某条历史数据时，根据该数据对应的“受检人当时年龄”和“受检人性别”，从该项目的所有适用范围中自动选择最匹配的一条（优先级：性别+年龄完全匹配 > 性别不限+年龄匹配 > 系统默认）。若无法匹配，则标记为“无匹配参考范围”，不标红但提示。
  - 数值型参考范围：支持“最小值-最大值”（如4.0-10.0）或单侧阈值（如<7.0）。
  - 定性/半定量：强制使用预设词库（阴性、阳性、弱阳性；半定量如Negative、Trace、1+、2+、3+、4+），并自动标准化。
- **化验单备注字段（新增）**
  - 在列映射阶段，允许用户指定一列作为“备注列”。
  - 备注内容属于非结构化文本，不参与数值计算，但会原样跟随数据行存储并展示。
  - 在趋势分析明细、异常核对页面，备注列始终可见，以灰色小字显示在数值下方或独立列。
  - 对于OCR识别出的整单备注（如检验医师签字、标本状态），系统提供单独的“整单备注”区域，用户可在数据确认时手动填写或从OCR文本中拖入。

#### 4.2 化验单录入与识别

##### 4.2.1 文件上传与OCR配额管理 (深化设计)
- **前置校验**：必须先选择“受检人”和“医院”，否则上传按钮禁用。
- **格式支持**：JPG、PNG、PDF，单文件≤10MB。
- **批量处理**：支持多文件拖拽上传，生成上传队列。
- **文件列表与状态管理（三栏式布局）**
  - 左侧为上传源，右侧为待处理队列。
  - 每条记录显示：缩略图、文件名、上传时间、状态标记。
  - **状态标记定义：**
    - `待识别`（灰色）：刚上传。
    - `识别中`（转圈）：调用OCR。
    - `识别失败`（红色）：可重试或手动录入。失败不计入OCR调用次数。
    - `待解析`（橙色高亮）：OCR成功，但未配置/应用列映射规则。
    - `数据待确认`（蓝色）：已完成列解析，等待核对入库。
    - `已入库`（绿色带对勾）：数据已写入。**重复上传相同文件（MD5校验）时直接提示“该文件已入库”，是否需要对应的化验单记录；如果用户确认需要，调出已保存的对应的化验记录让用户修改。**
- **OCR调用额度实时显示**
  - 位置：页面底部状态栏常驻，即使进入解析页面也可见。
  - 样式：仪表盘进度条。
  - 示例文本：“本月OCR剩余调用次数：45/200 次（已用155次）”
  - 颜色阈值：剩余>50绿；10-50橙；<10红，并弹出提醒“额度即将耗尽，请优先录入重要单据”。
  - 点击额度数字可查看明细：“本月已成功调用23次，调用失败2次”。
  - **接口调用成功就计入次数，调用失败不计入次数，不管ocr是否成功； 每个月月初次数置0； 数据维护页面需要可以让用户手工编辑条数以便和服务商报表校准**

##### 4.2.2 自定义解析工具（核心交互深化）
- **三步引导式操作流程：**
  - **第一步：选基准（粗筛表格区域）**
    - 左侧原始文件预览，右侧顶部提示“请框选表格区域”。
    - 系统自动识别表格线，用户可手动拖拽调整框选范围。
  - **第二步：表头映射（卡片流）**
    - 展示系统提取的标题行内容，以卡片形式排列。
    - 每张卡片下方为下拉选择框，需映射到以下系统字段：
      - **检测项目（必填）**
      - **结果数值（必填）**
      - **单位（可选/智能提取）**
      - **参考范围（可选/智能提取）**
      - **备注列（新增，可选）**：用户可将含文字说明的列映射为“备注”，如“镜检结果”、“标本备注”。
    - 如果表格内不含“采样时间”，则统一使用外部日期选择器指定。
  - **第三步：单元格级修复（微观操作）**
    - 进入智能网格视图，保留原始OCR文本悬浮提示。
    - **分割**：选中单元格，按空格/冒号/制表符分割，确认界面展示拆分前后对比。
    - **合并**：合并不同逻辑类型单元格时，弹出严重警告，并智能建议“是否拆分为多行？”。
    - 所有修改均支持Ctrl+Z撤销。
- **源数据留痕与历史对比**：在预览行侧展示近3次同项目历史数值（基于当前受检人）及当时年龄，辅助判断当前结果是否合理。

##### 4.2.3 批量数据录入与沉浸式比对（深化设计）
- **整体布局**：全屏模式，无折叠，左右固定分栏。
  - **左侧（40%宽度）**：原始凭证查看器。
    - 支持鼠标滚轮缩放（以指针为中心）、空格键+拖拽平移、旋转。
    - 与右侧数据联动：点击右侧数据行，左侧图片自动高亮对应行区域并居中。
  - **右侧（60%宽度）**：数据网格。
    - 表头冻结、首列冻结。
    - **列结构：**
      1. 状态图标列：异常时显示大红点（危极值闪烁），正常显示绿点；点击可筛选只查看异常项。
      2. 检测项目（冻结列）。
      3. 当前结果（加粗，异常数值红色）。
      4. 单位（灰色小字）。
      5. 本次参考范围（根据受检人年龄/性别自动匹配显示，格式如“4.0-10.0 (成年男性)”）。
      6. 历史趋势迷你图：3px高微折线，展示近3次结果，一目了然趋势。
      7. 备注列（新增）：显示映射的备注信息，如“镜下可见异形红细胞20%”，灰色小字，不干扰主要数据。
      8. 源数据追溯：眼睛图标，点击可跳转图片对应OCR截取位置。
  - **顶部吸顶工具栏**：`[保存并入库] [批量忽略异常] [导出CSV] [当前规则：XX医院-XX化验单]`，以及“仅查看异常项”开关，开启后正常行变淡但不消失。
- **数据入库确认**：用户核对无误点击保存，系统将数据写入数据库。如果发现遗漏备注，可随时返回编辑行级备注或整单备注。

---

#### 4.3 数据标准化与质控

##### 4.3.1 参考范围强制标准化与动态匹配
- 同4.1.3所述，建立多适用范围参考体系。
- 数据入库时，系统即刻计算“受检时年龄”并匹配适用参考范围，写入本条数据的“参考范围”字段。若项目无匹配范围，标记为“待补充参考范围”并向用户提示。
- 异常高亮实时基于匹配后的范围判断，而非固定默认范围。

##### 4.3.2 单位转换公式
- 自定义公式：用户可为医院规则配置单位转换公式（如mg/dL转mmol/L：x/18.018）。
- 公式验证：保存前强制要求输入示例值验证，系统输出结果供确认。

##### 4.3.3 OCR结果版本管理
- 记录每次保存的手动修正版本。后续同医院上传优先调用最新修正版作为模板。

##### 4.3.4 备注字段处理规则
- 行级备注：与数值一同存储，展示时始终紧邻数值。
- 整单备注：允许在数据预览页底部设置一个独立的“全单备注”文本框，可粘贴从OCR识别出的检验医师签名、标本状态等信息。存储为报告级别的备注。

---

#### 4.4 趋势分析与可视化

- **多源数据合并**：基于“化验项目别名”映射自动合并。
- **异常判定回溯**：图表中每个数据点的异常标记，依据该数据点发生时的受检人年龄、性别所匹配的参考范围动态重算，保证历史趋势的异常区间标注真实反映当时的标准。
- **图表交互**：
  - 支持单选、多选或全选项目。
  - 异常区间标注：折线图背景用色块表示正常参考带（因适用范围变化，参考带可能分段变化），超出区域标红。
- **明细查询**：点击数据点，下钻查看该时间点的化验单原始图片、完整明细、单位、当时参考范围及备注内容。
- **表格视图**：增加列“受检时年龄”、“参考范围（含适用范围说明）”、“备注”，表头支持排序和筛选。

---

### 5. 异常处理与边界场景

- **医院名称歧义**：输入模糊匹配，相似度>80%提示关联。
- **单位转换错误**：强制示例验证。
- **检测时间缺失**：解析配置阶段强制指定采样时间列，否则禁止保存。
- **参考范围缺失**：无匹配适用范围时，数据仍可入库但状态标记为“无参考”，并在界面上给出醒目提示（如黄底问号图标），引导用户去项目库补全该项目的适用范围配置。
- **备注列内容异常**：若OCR将数值误识别到备注列，系统在预览表格中提供一键“移动至数值列”的快捷按钮。

---

### 6. 非功能性需求

- **性能要求**：OCR识别单张图片<5秒，PDF（<10页）<15秒；数据预览渲染<2秒。
- **数据安全**：所有数据存储于本地sqlite，提供JSON/CSV导出备份。
- **兼容性**：仅适配最新版Chrome、Edge、Firefox，分辨率≥1024px。

---

### 7. 附录：术语定义

- **受检人**：数据归属主体，隔离维度。
- **标准术语**：系统内部统一使用的定性/半定量描述。
- **解析规则**：绑定医院维度的配置，包含列映射、单位转换、备注列指定等。
- **适用范围**：一条参考范围生效的性别和年龄区间，用于精准判读异常。
- **行级备注**：针对单条检测结果的文字补充说明。
- **整单备注**：针对整份化验单的备注信息（如标本状态、检验者注释）。
