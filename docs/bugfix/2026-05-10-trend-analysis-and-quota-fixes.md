# 2026-05-10 趋势分析与配额设置修复

## 修复内容

### 1. OCR月度配额设置优化
- **问题**：月配额校准界面允许用户修改总配额，但实际上总配额应在 `.env` 文件中配置
- **修复**：
  - 新增 `OCR_QUOTA_MONTHLY` 环境变量，用于配置月度配额上限
  - 设置页面改为允许用户校准当前用量（used_count）
  - 配额信息从环境变量读取，不再从数据库读取

### 2. 趋势分析明细列表修复
- **问题**：数值列显示不正确、单位列为空、置信度列不需要、提示符未计算、参考区间错误、趋势图显示错误
- **修复**：
  - **数值显示**：正确解析原始值，支持 `<`、`>` 等前缀（如 `<220` 解析为 `220`）
  - **单位显示**：使用 `COALESCE` 回退逻辑，优先使用 `normalized_unit`，其次 `original_unit`，最后 `test_items.default_unit`；去除尾部 `/` 字符
  - **移除置信度列**：从表格中移除不需要的置信度列
  - **提示符计算**：当没有 `ref_interval_id` 时，使用 `ref_interval_text` 解析参考区间并计算 H/L/normal
  - **参考区间显示**：优先使用 OCR 提取的 `ref_interval_text`，而不是可能错误的 `ref_interval_id` 查询结果
  - **趋势图修复**：按项目名称分组，单数据点显示为散点而非折线

### 3. 表格样式优化
- **列顺序调整**：采样日期 → 检验项目 → 数值 → 参考区间 → 提示符 → 单位 → 趋势 → 医院
- **列对齐**：所有列标题居中，数值/参考区间/提示符/单位居中，检验项目/趋势/医院靠左
- **视觉效果**：列间浅色虚线分隔，奇偶行交替底色

## 根本原因分析

1. **参考区间错误**：`test_item_id` 匹配逻辑导致多个项目被错误匹配到同一 ID（如中性粒细胞百分比和绝对值都匹配到 ID=9）
2. **flag 未计算**：当 `ref_interval_id` 为 NULL 时，无法计算 flag
3. **迷你趋势图分组错误**：使用 `test_item_id` 分组导致多个不同项目被合并显示

## 验证结果

- 嗜碱性粒细胞：0.11 > 0.10 → 正确显示 H
- 中性粒细胞：5.53 在 2.0-7.0 范围内 → 正确显示 normal
- D-二聚体：`<220` → 正确解析为 220
- 所有列标题居中显示
- 奇偶行交替底色
