#2
OCR API 不支持 PDF 文件（架构升级）
现象
上传 PDF 格式的检验报告后，OCR 状态变为 failed。
根因
旧代码调用阿里云 RecognizeGeneral（通用文字识别）接口，该接口不支持 PDF 格式输入。旧实现使用手动 HMAC-SHA1 签名构建 HTTP 请求，硬编码 Action: "RecognizeGeneral"。
修复
从手动签名 HTTP 调用迁移到阿里云 Go SDK v3，使用 RecognizeAllText 统一识别接口（Type=Advanced），原生支持 PDF 多页识别。
  • 移除 ~100 行手动签名代码（signRequest、HMAC-SHA1、URL 编码等）
  • 使用 github.com/alibabacloud-go/ocr-api-20210707/v3 官方 SDK
  • RecognizeAllText 通过 SubImages[].BlockInfo.BlockDetails 返回结构化文本块
  • 新增 Y 坐标分组 → Row 编号 → 按 X 排序的坐标提取逻辑

#4
左侧 PDF 文件预览不显示
现象
上传 PDF 后进入核效页面，左侧预览区域空白。
根因
<img> 标签无法渲染 PDF 文件。GetReportImage handler 直接用 c.File(filePath) 返回文件内容，浏览器收到 PDF 但 <img> 无法作为图片渲染。
修复
新增 isPdf 计算属性（检测 file_path 后缀），PDF 用 <embed> 渲染，图片继续用 <img>： 
<embed :src="reportImageUrl" v-if="reportImageUrl && isPdf"
       type="application/pdf" class="w-full border-0"
       style="height: 100%; min-height: 100%;">
高亮呼吸框和放大镜仅在图片模式下启用。

#5
右侧检验项目全部列为一行（解析器设计缺陷）
现象
OCR 识别完成后，核效页面右侧表格中所有文本每个单独成行（如"白细胞计数"、"5.2"、"10^9/L"、"3.5-9.5" 各占一行），没有分组为完整的检验项目。
根因
两个层面：
  1. OCR block → report_item 直接映射：旧 handler 对每个 OCR 文本块直接插入一条 report_items，没有将同一检验项目的名称/数值/单位/范围组合为一条记录。
  2. 状态机不支持两种字段顺序：化验单存在 名称→数值→范围→单位 和 名称→数值→单位→范围 两种排版，状态机遇到 unit 立即 emit，后面的 range 被丢弃。
修复
创建了 internal/services/ocr_parser.go，实现三级递进解析策略：
  • Path 1: 结构化解析 — 利用 OCR 坐标信息（Row + X），按行→列分组，每行内分类字段组合为 item
  • Path 2: 线性状态机 — 无坐标时的 fallback，新增双态支持（sValUnit / sValRng）
  • Path 3: 文本行级解析 — 当 block 包含整行文本时的最终 fallback
另外增加了：英文缩写库（50+）、小数合并、噪声过滤、文本分类器（name / value / range / unit / noise）。

#6
重新识别后右侧仍为空（解析器 fallback + 删除顺序缺陷）
现象
点击"重新识别"按钮后，OCR 状态变回 review，但核效页面右侧依然空白，无任何数据。
根因
复合问题，两个 bug 叠加：
  1. Bug 6a：阿里云返回的文本块可能是整行文本（如 "白细胞计数 5.2 10^9/L 3.5-9.5"），parseStructured 将整行分类为单个 "name"，parseLinear 两行都是 "name" 卡在 sName，parseContentText 用空格拼接多行导致分行错误。
  2. Bug 6b：ReOCR handler 中 DELETE FROM report_items 在解析前执行，若新版 OCR 返回 0 个文本块，旧数据已被删光而新数据为 0 → 永久空白。
修复
  • 修复 6a：ParseLabResults 入口增加检测，先扫描是否有"包含空格的整行文本块"，如有则跳过 parseLinear，直接用 "\n" 拼接传给 parseContentText
  • 修复 6b：ReOCR handler 新增 len(ocrResults) == 0 检查，零结果时不删除旧数据、标记 failed 返回；DELETE 移到确认有新数据后的位置；Upload goroutine 同样添加零结果保护
